Resumo:

O 7º trabalho prático da unidade curricular de Desenvolvimento de Aplicações Web tem como objetivo a criação de uma aplicação que recebe ficheiros carregados pelo utilizador/cliente e os armazena numa diretoria no lado do servidor. A aplicação, para além de permitir o upload de ficheiros, deve mostrar ao utilizador os ficheiros já carregados e guardados no servidor.

Tal como tem vindo a acontecer nos trabalhos para casa anteriores da disciplina, o principal foco é a programação de um servidor. Esta iteração coloca em uso vários pacotes que facilitam ou simplificam o processo de programação que tem vindo a ser utilizado. Entre estes encontram-se a framework Express, o módulo de parsing de dados de formulários Formidable, o logger de pedidos HTTP Morgan e o já utilizado motor de templates Pug.

As funcionalidades de carregamento e listagem de ficheiros são suportadas tanto por páginas individuais (uma página para o formulário, com o path /form e uma página para a listagem, com o path /ficheiros), como por uma página que combina as duas funcionalidades (servida em /). A página única que combina as duas funcionalidades funciona ainda sem recurso a jQuery e AJAX, havendo portanto o envio dos dados submetidos para o servidor (ficheiro carregado e uma pequena descrição) e uma nova geração completa da página, seguida do seu envio para o cliente. Tal como já foi referido, os ficheiros enviados são guardados numa diretoria no servidor e o seu nome e a sua descrição ficam registados num ficheiro JSON, sobre a forma de um lista de objetos (um por ficheiro), que suporta a funcionalidade de listagem. Esta funcionalidade, por sua vez, providencia ligações para os ficheiros carregados e a partir dessas há suporte para servir os ficheiros carregados ao cliente.
