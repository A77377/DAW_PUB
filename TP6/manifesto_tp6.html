Resumo:

Para o 6º trabalho prático da unidade curricular de Desenvolvimento de Aplicações Web é necessário criar uma aplicação de registo de tarefas do quotidiano.

Na base da aplicação encontra-se um servidor, semelhante aos realizados para os trabalhos anteriores, que responde adequadamente a mensagens de 2 métodos do protocolo HTTP (GET e POST, em particular) e que gera páginas HTML com recurso ao motor de templates Pug. Novamente, o servidor analisa o path contido no URL do pedido para encontrar a página pretendida, no caso de se encontrar em uso um caminho previsto. Todas as páginas servidas são geradas a partir de templates pelo motor Pug.

Uma das novidades para esta iteração dos trabalhos práticos é o uso de formulários, para preenchimento de informação a registar. No caso particular desta aplicação, os formulários são referentes a tarefas a cumprir, devendo ser submetidos - pelo uso do método HTTP POST, outra das novidades - data de registo, prazo de cumprimento, descrição e tipo da tarefa, havendo também espaço para outras possíveis adições (como o grau de prioridade da tarefa, por exemplo). Outra das novidades é a introdução da persistência de dados, que aqui se efetua de forma simples pelo uso do pacote jsonfile, com o qual se regista num ficheiro JSON válido os dados submetidos através da aplicação. Está também prevista a funcionalidade de remoção de tarefas.
